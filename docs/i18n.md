# i18n — Conventions

Library: `next-intl` · Default locale: `en` · Additional: `de`

---

## URL structure

Strategy: `as-needed` — default locale has no prefix, non-default gets `/de/...`.

| Route | English | German |
|---|---|---|
| Landing | `/` | `/de` |
| Login | `/auth/login` | `/de/auth/login` |
| Dashboard | `/app` | `/de/app` |

No `/en/` URL is ever emitted. If you see `/en/...` something is misconfigured.

---

## File locations

```
messages/en.json    ← English (default)
messages/de.json    ← German
src/i18n/routing.ts    ← locale list, prefix strategy
src/i18n/navigation.ts ← locale-aware Link / redirect / usePathname / useRouter / getPathname
src/i18n/request.ts    ← server config (getRequestConfig)
src/proxy.ts        ← next-intl routing + auth gate (Next 16 proxy convention)
```

> The matcher excludes the file-based metadata routes (`icon`, `apple-icon`,
> `opengraph-image`, `sitemap.xml`, `robots.txt`) alongside `favicon.ico` — they
> generate extension-less URLs, so without the exclusion auth+intl redirect the
> browser's tab-icon request to `/auth/login` and the PNG never renders.

> ℹ **Migrated 2026-07-05:** `middleware.ts` → `proxy.ts` (pure rename per
> `nextjs.org/docs/messages/middleware-to-proxy`). The suspicion about the `auth()`
> HOF proved right — wrapping the proxy in it turns next-intl's default-locale
> rewrite into an infinite 307 loop in the standalone production server (dev is
> unaffected). The proxy therefore reads the session via `getToken()` directly;
> see the docblock in `src/proxy.ts` before changing this.

## Namespace structure

Nested, not flat. Format: `area.sub.key`

```
common.brand.*           name, tagline
common.actions.*         signIn, signUp, getStarted, …
common.nav.*             features, pricing, docs
landing.hero.*
landing.features.*
landing.pricing.*
landing.cta.*
landing.footer.*
auth.signIn.*
auth.signUp.*
auth.errors.*            invalidEmail, passwordRequired, …
auth.oauth.*             (kept for future OAuth; currently unused)
language.*               en, de, switcher.label
dashboard.*              overview, kpi, range, chart, activity, …
invoices.*               status, statusOptions, form, detail, storno, draftNumber, …
app.*                    nav, user, theme, pages, settings.* (tabs: profile, sender, tax, security, …)
```

## Usage

**Server component:**
```ts
import { getTranslations } from "next-intl/server";
const t = await getTranslations("landing.hero");
```

**Client component:**
```ts
import { useTranslations } from "next-intl";
const t = useTranslations("auth.signIn");
```

**Arrays / raw values:**
```ts
t.raw("plans.pro.features") as string[]
```

## Formatters

Helpers in `src/lib/i18n/formatters.ts`:
- `useFormatCurrency()` — always EUR for MVP, locale-aware symbol position
- `useFormatDate()` — medium date style
- `useFormatRelative()` — "3 days ago" / "vor 3 Tagen" via `date-fns` locale

## Linguistic tone

- **German:** "Sie" form. "Ihre Rechnungen", never "Deine Rechnungen".
- **English:** natural business English.
- No marketing speak inside the product.

---

## Status label translations

All statuses must be translated in both files, keyed by API status name under `invoices.status.*`:

| Status (API) | EN label | DE label |
|---|---|---|
| Draft | Draft | Entwurf |
| Finalized | Open | Offen |
| Paid | Paid | Bezahlt |
| Overdue *(derived — `isOverdue` / virtual filter, not stored)* | Overdue | Überfällig |
| Cancelled | Cancelled | Storniert |

`Sent` / `Versendet` no longer exists — it was folded into `Finalized`, which is labelled "Open" / "Offen" in the UI.

Draft invoices have no invoice number yet — the number cell shows `invoices.draftNumber` ("Draft" / "Entwurf") instead.
